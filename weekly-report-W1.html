整體背景

MDMMobile Device Management，行動裝置管理，遠端納管與管控設備的系統
W1 / W2五週開發計畫的第 1 / 第 2 週
協議層設備與伺服器通信協議的實作層

後端與資料

DB schema資料庫的資料表結構設計
tenant租戶；多租戶架構中的一個客戶／機構，資料彼此隔離
device_group / device「設備分組」與「設備」資料表
profiles / apps / webhooks / audit配置檔／應用／事件回呼／稽核日誌等資料表
API應用程式介面，系統間溝通的接口
CRUD建立 / 讀取 / 更新 / 刪除四種基本操作
鑑權身分驗證與權限授權

事件推送（Webhook）

Webhook事件回呼；系統發生事件時主動 HTTP 推送通知對方
HMAC 簽名用密鑰對訊息算出簽名，防偽造與竄改
三段退避重試推送失敗後按 30 秒 → 5 分 → 30 分 遞增間隔重試
死信佇列Dead Letter Queue；重試都失敗後留底的訊息佇列，可日後補推

Windows MDM 協議與配置

CSPConfiguration Service Provider；Windows 內建的配置介面，每個 CSP 管一類設定
Registry CSP操作 Windows 登錄檔（Registry）的 CSP
EDA-CSPEnterpriseDesktopAppManagement CSP；遠端派發 .msi 桌面應用
OMA-DMOpen Mobile Alliance Device Management；標準設備管理協議，命令下發的基礎
ackacknowledge；設備回傳的「命令已執行」確認回執
Profile 引擎把配置（Profile）下發到設備並追蹤套用狀態的模組

Agent App 與打包

Agent App安裝在設備上的客戶端代理程式
.NET 8 Service用微軟 .NET 8 框架寫的 Windows 背景服務
腳手架scaffold；專案的初始骨架／框架
WiX把程式打包成 Windows 安裝包（.msi）的工具
.msiWindows 標準安裝包格式
簽名code signing；用憑證數位簽署，證明來源可信、未被竄改
install-agent一鍵把 Agent 派發到設備的 API
enrollment設備納管／註冊進 MDM 接受管理
配置注入把 MDM 位址等設定寫入設備（HKLM 登錄檔）
Reboot 命令遠端重啟設備的命令
單元測試針對單一函式／模組的自動化測試

規格文檔

ID Design Spec各類 ID（租戶／設備／命令…）的格式與生命週期規範
API SpecificationAPI 規格：鑑權、錯誤格式、分頁、限流
Webhook SpecWebhook 事件清單、payload、簽名、重試規格
Agent API SpeciOS ＋ Windows Agent 共用的上報 API 規格
Data Model & State Machine資料模型與狀態機（設備／命令／配置的狀態流轉）
OpenAPI 3.0描述 REST API 的業界標準格式，可自動產生文件與 client SDK
